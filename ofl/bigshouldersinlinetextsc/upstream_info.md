## Override config.yaml

An override `config.yaml` existed in the google/fonts family directory from the initial onboarding commit. This config was necessary because the SC variant did not exist as a pre-built binary in the upstream repo. Instead, the override config used gftools-builder recipe operations to:

1. Build the variable font from `BigShouldersInline.glyphs`
2. Subspace the optical size axis to `opsz=10` (text variant)
3. Apply `remapLayout` to remap `smcp -> ccmp` (producing the small caps variant)
4. Rename the family to "Big Shoulders Inline Text SC"

The `source_file` entry `Big-Shoulders-Inline/fonts/variable/text/BigShouldersInlineTextSC[wght].ttf` in METADATA.pb pointed to a file that did not actually exist in the upstream repo at the referenced commit. The font was built from sources using the override config, not copied from a pre-built binary. Only `BigShouldersInlineText[wght].ttf` (without SC) existed in that directory.

Since the override config.yaml was present in the google/fonts family directory, the `config_yaml` field was correctly omitted from METADATA.pb (google-fonts-sources auto-detects local overrides).

## Note on the `source_file` Entry

The METADATA.pb `files` block references `Big-Shoulders-Inline/fonts/variable/text/BigShouldersInlineTextSC[wght].ttf` as a `source_file`, but this file did not exist in the upstream repo at commit `0b3d09a`.
