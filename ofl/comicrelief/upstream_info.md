## fontc_crater Build Fix (2026-05-21)

**Model**: Claude Opus 4.7

### Initial state
METADATA.pb pointed `config_yaml` at the upstream `sources/config.yaml`. fontc_crater failed with `missing source 'ComicRelief-Bold.ufo'`.

### Investigation
The recorded commit `856315f` is correct (tag `v1.2`, matching the `archive_url`). The upstream `sources/config.yaml` lists `.ufo` sources, but the repository at that commit contains only FontForge `.sfd` files (`sources/ComicRelief-Regular.sfd`, `sources/ComicRelief-Bold.sfd`). The `.ufo` files are build artifacts generated by the repository's `make convert` step (`sfd2ufo`); they were never committed at any point in the repo history. gftools-builder (and fontc_crater) cannot run the `sfd2ufo` conversion, so the config can never be satisfied as-is.

### Actions taken
The non-functional `config_yaml: "sources/config.yaml"` field was removed from the METADATA.pb `source` block. No override `config.yaml` was created, because the upstream sources are FontForge `.sfd` files requiring a non-gftools conversion step. The repository URL, commit and `archive_url` are correct and were left unchanged.

### Final state
METADATA.pb retains the correct repository URL and commit (`v1.2`); the misleading `config_yaml` reference is gone. The family is recorded as not reproducible by gftools-builder — its upstream sources are FontForge `.sfd` files, not gftools-builder compatible without a separate conversion step.
