## Summary

Libre Barcode 39 Text is a barcode font designed by Lasse Fister (graphicore) for rendering Code 39 barcodes with human-readable text below the bars. It is part of the broader Libre Barcode project, which includes fonts for Code 39, Code 39 Extended, Code 128, and EAN-13 barcode standards. The upstream repository was confirmed at `https://github.com/graphicore/librebarcode`, and the binary in google/fonts was verified as an exact match to the upstream binary at the referenced commit.

## Build System

The Libre Barcode project used a **custom JavaScript/Node.js build system**, not gftools-builder. The build pipeline worked as follows:

1. **Node.js barcode generator** (`app/bin/build` and `app/lib/builder/`) programmatically generated UFO sources by encoding barcode glyphs using JavaScript (with barcode-standard-specific logic in `code39.js`, `code128.js`, `ean13.js`, etc.)
2. **fontmake** compiled the generated UFO sources into TTF fonts
3. **gftools fix-font** applied post-processing fixes
4. The **`app/bin/buildAll`** shell script orchestrated the full pipeline

Build dependencies included Node.js with npm, bower, Python with fontmake/gftools (from `requirements.txt`), and ttfautohint.

**There was no `config.yaml` in the repository**, and one could not be created as a gftools-builder override because the font sources were procedurally generated by JavaScript code rather than existing as static `.glyphs`, `.ufo`, or `.designspace` files. The build process was fundamentally incompatible with gftools-builder.

### Upstream Commit Analysis

The commit `f9864c42` (2020-12-16, "[ean13] bump version add binary v1.008") did not directly modify `LibreBarcode39Text-Regular.ttf`. The font file was last modified in the immediately preceding commit `63e8744e` (2020-12-16, "[code39/128] add binaries v1.005"), made just 1 minute earlier by the same author. Both commits were part of the same release session:

- `63e8744` (01:39:13 UTC+1) — Added Code 39 and Code 128 binaries v1.005 (modified `LibreBarcode39Text-Regular.ttf`)
- `f9864c4` (01:40:33 UTC+1) — Added EAN-13 binary v1.008 (did NOT modify `LibreBarcode39Text-Regular.ttf`)

Since `f9864c4` came after `63e8744` and the font file was unchanged between the two commits, the `f9864c4` reference was correct — it was the HEAD of master when the packager fetched the font. After `f9864c4`, only two more commits were made on master:
- `93bb140` (2022-04-18) — EAN-13 bulk encoder feature
- `e44a85e` (2024-06-25) — Bug fix for a JS variable

Neither of these later commits modified any font binary files.
